Projet Hardening Debian 12

Installation et sécurisation complète d'un système Debian 12 avec chiffrement LUKS, configuration AppArmor et durcissement système

Présentation du Projet

Ce projet consiste en l'installation et la sécurisation complète d'un système Debian 12 suivant les meilleures pratiques de cybersécurité. L'objectif était de créer un système durci contre les attaques, en implémentant plusieurs couches de sécurité.

Debian 12 LUKS Encryption AppArmor UFW Firewall Fail2Ban Nftables Docker Rootless

Configuration Initiale

Configuration matérielle : Machine virtuelle avec 8GB RAM et configuration réseau bridgée

Spécifications VM

  • RAM : 8 GB
  • Processeurs : 4 cœurs
  • Stockage : 100 GB avec chiffrement
  • Réseau : Mode bridgé sécurisé

Sécurité Hardware

  • Désactivation des ports USB non utilisés
  • Protection contre les attaques DMA
  • Configuration UEFI sécurisée
  • Mot de passe BIOS obligatoire

Partitionnement Sécurisé - Architecture LVM sur LUKS

Schéma de partitionnement : Configuration LVM sur LUKS avec chiffrement AES-256 complet

Architecture de chiffrement

Configuration complète avec chiffrement LUKS AES-256 et gestion LVM pour une sécurité maximale. Le volume chiffré contient tous les systèmes de fichiers sauf /boot.

Partition BOOT (Non chiffrée)

  • Taille : 1 GB
  • Type : Primaire - ext2
  • Point de montage : /boot
  • Options : relatime, nodev, nosuid, noexec
  • Sécurité : Empêche l'exécution et l'escalade privilèges

Partition LUKS (Chiffrée)

  • Taille : 99 GB (espace restant)
  • Chiffrement : AES-256 bits
  • Algorithme IV : xts-plain64
  • Méthode : dm-crypt avec phrase secrète
  • Effacement : Données préexistantes effacées

Volumes logiques LVM dans le conteneur chiffré

Groupe de volumes

Volume ROOT (/) : 30 GB - ext4

• Options: défaut, blocs réservés 1%

• Système de fichiers principal durci

Volume VAR (/var) : 10 GB - ext4

• Options: relatime, nodev, nosuid

• Protection contre périphériques et escalade privilèges

Volume LOG (/var/log) : 10 GB - ext4

• Options: relatime, nodev, nosuid, noexec

• Isolation complète des logs système

Volume DOCKER (/var/lib/docker) : 50 GB - xfs

• Options: relatime, nodev, nosuid, noexec

• Système de fichiers optimisé pour containers

Volume HOME (/home) : Espace restant - ext4

• Options: relatime, nodev, nosuid, noexec

• Protection maximale des données utilisateur

Options de montage sécurisées

  • relatime : Réduit les écritures d'accès
  • nodev : Bloque les fichiers périphériques
  • nosuid : Ignore les bits SUID/SGID
  • noexec : Empêche l'exécution de binaires

Avantages sécuritaires

  • Chiffrement complet des données sensibles
  • Isolation des composants système
  • Prévention d'escalade de privilèges
  • Protection contre l'exécution malveillante

Procédure de configuration LUKS

Configuration du chiffrement : Mise en place du volume chiffré avec effacement sécurisé

Étapes de configuration du chiffrement

Procédure LUKS + LVM

1. Configurer les volumes chiffrés → Créer des volumes chiffrés

2. Sélectionner la partition désignée pour LUKS (99 GB)

3. Effectuer l'effacement complet des données (recommandé)

4. Définir une phrase secrète robuste pour le chiffrement

5. Configurer device-mapper comme "Volume physique pour LVM"

6. Créer le groupe de volumes "starfighter"

7. Créer chaque volume logique avec ses spécifications

Vérification du Chiffrement

Disque chiffré opérationnel : Confirmation que le système démarre correctement avec le chiffrement LUKS actif et les volumes montés

Validation du chiffrement

Le système démontre que le chiffrement LUKS fonctionne parfaitement. Au démarrage, la phrase secrète est demandée pour déverrouiller le volume chiffré, et tous les systèmes de fichiers sont correctement montés dans leurs partitions logiques respectives.

Configuration AppArmor

AppArmor actif : Profils de sécurité pour tous les services critiques

Profils AppArmor

  • Protection des navigateurs web
  • Isolation des services système
  • Contrôle d'accès aux fichiers
  • Limitation des capacités réseau

Règles de sécurité

  • Deny /usr/bin/bash ix (Shell restreint)
  • Deny /etc/shadow r (Protection mots de passe)
  • Allow network rules seulement pour services autorisés
  • Restriction accès fichiers sensibles

Pare-feu et Protection Réseau

Configuration nftables : Structure des chaînes de filtrage avec politique DROP par défaut et règles spécifiques pour HTTP, HTTPS, DNS, NTP

Configuration UFW + Fail2Ban

Règles de sécurité

Policy INPUT: DROP (Blocage par défaut)

Policy FORWARD: ACCEPT

Policy OUTPUT: ACCEPT

Ports autorisés: 80, 443, 53, 123 (HTTP, HTTPS, DNS, NTP)

Fail2Ban: Bannissement automatique après 3 tentatives

Durcissement Système

Kernel Hardening

Configuration sysctl avec désactivation IPv6, protection contre SYN flood, et ASLR activé.

Services Minimaux

Désactivation de tous les services non essentiels et suppression des paquets inutiles.

Docker Rootless

Installation Docker en mode rootless pour éviter les privilèges root.

Mises à jour Auto

Configuration unattended-upgrades pour les mises à jour de sécurité automatiques.

Mises à jour automatiques

Configuration unattended-upgrades : Mises à jour de sécurité automatiques avec nettoyage des anciens noyaux et reporting

Automatisation de la sécurité

Paramètres configurés

InstallOnShutdown: false (préservation de la disponibilité)

MailReport: "on-change" (notifications ciblées)

Remove-Unused-Kernel-Packages: true (nettoyage automatique)

Remove-New-Unused-Dependencies: true (optimisation espace)

Automatic-Reboot: false (contrôle manuel des redémarrages)

Configuration réseau sécurisée

Durcissement réseau : Désactivation des redirections ICMP et configuration sysctl pour la sécurité réseau

Paramètres sysctl

  • Désactivation IPv6 (si non utilisé)
  • Protection anti-spoofing (rp_filter)
  • SYN cookies activés
  • Redirections ICMP désactivées

Sécurité additionnelle

  • ASLR activé (randomize_va_space=2)
  • Core dumps sécurisés
  • Protection des liens symboliques
  • Logging des paquets suspects
