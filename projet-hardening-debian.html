Chiffrement LUKS - Protection des Données

Architecture sécurisée : LVM sur LUKS avec chiffrement AES-256

Principe du Chiffrement

LUKS (Linux Unified Key Setup) chiffre l'intégralité du disque avec un algorithme AES-256 bits. Même en cas de vol physique, les données restent illisibles sans la phrase secrète.

Partition Boot (Non chiffrée)

Taille : 1 GB
Système : ext2
Options de sécurité :
  • • noexec : empêche l'exécution de programmes
  • • nosuid : ignore les permissions spéciales
  • • nodev : bloque les fichiers périphériques
  • • relatime : réduit les écritures sur le disque

Volume Chiffré LUKS

Taille : 99 GB (reste du disque)
Chiffrement : AES-256 bits
Contenu :
  • • Système racine (/) - 30 GB
  • • Répertoire variable (/var) - 10 GB
  • • Logs système (/var/log) - 10 GB
  • • Données Docker (/var/lib/docker) - 50 GB
  • • Données utilisateur (/home) - Reste de l'espace

Configuration LUKS : Création du volume chiffré avec effacement sécurisé

AppArmor - Contrôle d'Accès Obligatoire

AppArmor opérationnel : Profils de sécurité actifs pour tous les services

Qu'est-ce qu'AppArmor ?

AppArmor limite les actions que peuvent effectuer les programmes. Même si un logiciel est compromis, il ne peut accéder qu'aux ressources autorisées par son profil de sécurité.

Protections Actives

Navigateurs web

Limitation des accès fichiers et réseau

Services système

Isolation des processus critiques

Applications utilisateur

Restriction des capacités privilégiées

Exemples de Règles

# Interdictions

deny /usr/bin/bash ix

deny /etc/shadow r

# Autorisations limitées

allow network inet

allow /home/user/** rw

Protection Réseau - NFtables et Fail2Ban

Pare-feu NFtables : Règles de filtrage avec politique restrictive par défaut

Stratégie de Sécurité

Principe : Bloquer tout par défaut, autoriser uniquement le nécessaire. Seuls les ports essentiels (HTTP, HTTPS, DNS, NTP) sont ouverts.

Configuration UFW + Fail2Ban

# Politique par défaut

INPUT: DROP (bloquer toutes les connexions entrantes)

OUTPUT: ACCEPT (autoriser les connexions sortantes)

# Ports autorisés

80/tcp (HTTP) - Sites web

443/tcp (HTTPS) - Sites web sécurisés

53/udp (DNS) - Résolution de noms

123/udp (NTP) - Synchronisation temps

# Fail2Ban: Bannissement après 3 tentatives
