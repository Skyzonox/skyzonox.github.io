Projet Administration Infrastructure

Déploiement et sécurisation d'une infrastructure de gestion de parc informatique avec GLPI et DynFi Firewall

Présentation du Projet

Ce projet consiste en la mise en place d'une maquette fonctionnelle de gestion de parc informatique, intégrant GLPI pour l'administration des actifs IT et DynFi Firewall pour la sécurisation réseau. L'objectif était de créer une infrastructure complète avec segmentation réseau et règles de sécurité adaptées.

GLPI 10.0.7 DynFi Firewall Linux Debian Apache2 MariaDB PHP Configuration Réseau

Architecture Réseau Déployée

Architecture Multi-Zones : Segmentation WAN/LAN/DMZ pour une sécurité optimale

Configuration Réseau

  • WAN (em0) : Interface Internet en DHCP
  • LAN (em1) : Réseau interne 10.0.0.252/24
  • DMZ (em2) : Zone serveurs 192.168.1.1/24
  • Isolation : Séparation complète des zones

Règles de Sécurité

  • Blocage total des flux DMZ vers LAN
  • Autorisation HTTPS/HTTP/ICMP/SSH LAN→DMZ
  • Accès HTTP/HTTPS uniquement WAN→DMZ
  • Filtrage par protocoles et ports

Cahier des Charges Implémenté

Spécifications de sécurité : Règles ACL pour la sécurisation de l'infrastructure

Règles de Sécurité Appliquées

Sécurisation par ACL :

• Blocage des flux DMZ → LAN

• Autorisation HTTPS, HTTP, ICMP, SSH depuis LAN vers DMZ

• Autorisation HTTP et HTTPS uniquement depuis WAN vers DMZ

Installation et Configuration GLPI

Étape 1 : Environnement LAMP

Commandes d'installation

sudo apt update && sudo apt upgrade -y

sudo apt install apache2 mariadb-server -y

sudo apt install php php-mysql php-xml php-mbstring php-curl php-gd php-intl php-zip php-bz2 php-ldap php-imap php-apcu php-xmlrpc -y

Étape 2 : Configuration Base de Données

Configuration MariaDB

CREATE DATABASE glpi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER 'glpi'@'localhost' IDENTIFIED BY 'mot_de_passe';

GRANT ALL PRIVILEGES ON glpi.* TO 'glpi'@'localhost';
