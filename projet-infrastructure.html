Administration Infrastructure

Déploiement d'une infrastructure complète avec GLPI pour la gestion de parc informatique et DynFi Firewall pour la sécurité réseau

GLPI DynFi Firewall Debian Apache / MariaDB

Contexte du Projet

Objectif : Créer une infrastructure complète pour gérer un parc informatique d'entreprise.

Solutions déployées :

  • • GLPI : Gestion des équipements, tickets d'incidents et inventaire
  • • DynFi Firewall : Sécurisation des flux réseau et segmentation
  • • Architecture 3 zones : WAN, LAN et DMZ pour isoler les services

Architecture Technique

Schéma réseau : Segmentation en trois zones distinctes pour optimiser la sécurité

Configuration Réseau

Zone WAN (em0)

Connexion Internet via DHCP

Zone LAN (em1)

Réseau interne 10.0.0.252/24 pour les utilisateurs

Zone DMZ (em2)

Réseau serveurs 192.168.1.1/24 pour GLPI

Règles de Sécurité

BLOCAGE

DMZ → LAN (isolation complète)

AUTORISÉ

LAN → DMZ (HTTP, HTTPS, SSH, ICMP)

LIMITÉ

WAN → DMZ (HTTP et HTTPS uniquement)

Déploiement GLPI

Étape 1 : Environnement Serveur

Installation des composants nécessaires :

# Mise à jour du système

apt update && apt upgrade

# Installation LAMP

apt install apache2 mariadb-server php

Étape 2 : Base de Données

Configuration MariaDB pour GLPI :

CREATE DATABASE glpi;

CREATE USER 'glpi'@'localhost';

GRANT ALL ON glpi.* TO 'glpi'@'localhost';

Interface GLPI opérationnelle : Dashboard de gestion avec statistiques temps réel

Console DynFi : Interface de gestion centralisée du pare-feu

Résultats Obtenus

Infrastructure Opérationnelle

GLPI fonctionnel avec base de données sécurisée, accessible depuis le réseau LAN avec toutes les fonctionnalités de gestion d'actifs.

Sécurité Réseau

Segmentation réseau effective avec règles de filtrage strictes entre les zones WAN, LAN et DMZ.

Gestion Centralisée

Interface unique pour l'administration complète du parc informatique avec système de tickets intégré.

Monitoring Temps Réel

Surveillance continue des connexions réseau et analyse détaillée du trafic via l'interface DynFi.
